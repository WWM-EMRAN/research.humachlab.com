Project Philosophy

Every project should create at least one visible research, software, or learning output.

HUMACH projects are organised as practical pathways where research questions become experiments, development tasks become prototypes, and student participation becomes measurable portfolio evidence.

Research outputs

Papers, reports, datasets, evaluation protocols, and reproducible experiments.

Development outputs

Web tools, dashboards, prototypes, scripts, and open-source-ready assets.

Training outputs

Internship tasks, supervised mini-projects, technical documentation, and portfolio items.

Collaboration outputs

Shared project briefs, team workflows, partner-facing demos, and impact stories.

Workflow

How a HUMACH project progresses

01

Define

Clarify the research question, target user, dataset, tool, or learning objective.

02

Build

Create experiments, prototypes, scripts, dashboards, documentation, or model pipelines.

03

Evaluate

Test with meaningful metrics, feedback, reproducibility checks, and project review.

04

Share

Convert work into papers, demos, reports, GitHub repositories, or student portfolios.
